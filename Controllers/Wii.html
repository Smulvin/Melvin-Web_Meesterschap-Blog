Opdracht

Het doel van dit project is dat je een persoonlijke website moest maken. Op deze website moest in ieder geval je leerdoelen staan en een gedeelte uit de content API van school gehaald worden met studenten informatie. Ik had als eerste idee om meerdere controllers te maken voor een repsonsive idee. Later had ik dit gewisseld naar een wii controller, omdat je die verticaal en horizontaal kan gebruiken.

Leerdoel 1:

Ik wil mezelf verbeteren in het maken van animaties door gebruik van CSS, zodat ik mijn websites er interactiever en leuker uit kan laten zien.

Ik heb dit leerdoel voornamelijk gewerkt tijdens mijn CSS website. Daar waren we namelijk gelimiteerd tot alleen HTML en CSS, maar nog steeds veel interactie en beweging. Daarom probeerde ik daar extra aandacht te besteden aan animatie, ook gewoon een paar simpele. Ik vindt dat ik hier flink op vooruit ben gegaan. OOk op mijn API website heb ik een kleine extra animatie toegevoegd als feedback naar de gebruiker wanneer hij een pokemon aan zijn team toevoegt.

Leerdoel 2:

Ik wil mijn website correct responsive maken door gebruik te maken van de juiste styling in CSS en waar mogelijk zonder media queries. Zodat mijn website er op alle schermen er goed uit blijft zien.

Dit leerdoel heb ik op eigenlijk elke website wel geprobeerd te maken. Zo heb ik ook op mijn sprint 0 website op een creatieve manier de website responsive gemaakt. Maar op andere websites zoals de detailpagian van API was het voor me gevoel niet mogelijk zonder media queries, maar dat kwam vooral de styling die ik had. Door een bevel cornershape was het lastig omdat tekst steeds werd afgesneden. Maar ook met mijn BT website was de website volledig responsive.

Leerdoel 3:

Ik wil meer schetsen en ideeën uitwerken op papier, zodat ik een beter beeld heb van het project voordat ik ga beginnen met coderen.

Ik heb aan dit leerdoel tijdens verschillende projecten gewerkt. Ik had er het meeste aan tijdens mijn API website. Dit gaf me namelijk een goed beeld van de opdracht en de gehele website. Hierdoor kon ik makkelijk componeten eruit halen die terugkwamen op verschillende pagina's. Maar ook voor dit blog heb ik het veel gebruikt. Met elke schets kreeg ik een steeds beter beeld van de controller die ik aan het maken was. Want ik wist steeds beter tijdens het schetsen waar ik op moest letten.

Reflectie

Ik merkte tijdens het maken van mijn sprint 0 website hoeveel oog voor detail ik heb en hoe leuk ik het vind om echte voorwerpen na te maken met code. Maar iets anders wat me op viel is hoe leuk het is om easter eggs toe te voegen. Voor vorige projecten tijdens mijn studie was de focus altijd gewoon een goed product, maar tijdens deze minor wordt het veel aangeraden. Ik mekrte hoe het leuk is dat als je een gek idee hebt, om het gewoon toe te voegen. Ik heb in elke opvolgend project wel een paar easter eggs verstopt op de website.
