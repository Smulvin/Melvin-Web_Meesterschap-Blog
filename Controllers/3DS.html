API

Opdracht

Voor dit vak moest ik een website maken die gebruikt maakte van 1 content API en 2 web APIs. Oorspronkelijk wilde ik iets met een canvas en een gamepad doen, maar er werd mij aangeraden om de AR API te gebruiken, later werd dit een library vanwege browser restricties.

Easter Eggs

Tijdens het maken van dit project kon ik leuk spelen met easter eggs, Zo zijn er sfx toegevoegd om het meer een game te laten lijken en zitten er toetscombinaties op de website voor leuke andere sfx.

Leerdoelen

Tijdens dit vak heb ik veel kunnen werken aan mijn leerdoeln. Ik heb aan mijn tweede leerdoel kunnen werken, het goed responsive maken. Voor veel onderdelen zoals de overview ging het vrij vanzelf. Maar in de detailpagina heb ik daar veel aandacht aan besteed. Het was namelijk lastig vanwege de styling die ik had. Uiteindelijk lukte het niet om dit volledig zonder media quereis te doen.

Leerdoelen

Ook kon ik tijdens dit vak goed werken aan mijn derde leerdoel. Zo heb ik voor mijn derde leerdoel mooi eerst schetsen kunnen maken voordat ik begon met coderen. Dit gaf me een goede houvast van hoe elke pagina eruit kwam te zien. Zoals de header en de footer als constante componenten.

Reflectie

Aan dit vak had ik veel aandacht besteed. Ik vond het leuk om aan dit project te werken omdat het een onderwerp dat ik zelf kon uitkiezen. Helaas had ik te veel focus op het leuk maken en minder op netheid. De code was ingewikkeld vanwege de AR library en het werken met API's, hierdoor verloor ik overzicht. Hierdoor was mijn code niet heel robust en netjes. Daar moet ik met een volgend project beter op letten.
